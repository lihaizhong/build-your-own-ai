# Agent智能体系统的设计与应用

提示词 + 插件 = Agent

---

反应式架构（Reactive Architecture）

快速决策的“直觉型”智能体

- 感知
- 决策
- 执行

优点：

- 速度快
- 简单可靠

缺点：

- 缺乏适应性
- 短视性

---

深思熟虑智能体（Deliberative Agent）

特点：基于内部模型进行规划，通过推理选择最优行动方案，具有长期目标导向性。

核心流程：

- 感知
- 建模
- 推理
- 决策

优点：

- 能处理多步复杂任务
- 优化长期目标而非即时反馈
- 适应动态变化环境

---

混合智能体架构（Hybrid Agent Architecture）

特点：结合反应式的“快速本能”和深思熟虑的“战略规划”，实现智能与效率的平衡。

- 底层（反应式）
  - 处理简单直接的查询，如市场状况、账户信息等
  - 毫秒级响应速度，提供快速反馈
  - 基于预设规则和简单逻辑作出决策
- 中层（协调）
  - 评估任务类型和优先级
  - 动态选择处理模式（反应式或深思熟虑）
  - 管理系统资源分配
- 顶层（深思熟虑）
  - 处理复杂的投资分析和长期财务规划
  - 多步骤、深度思考过程
  - 构建内部模型并生成多个备选方案

---

构建 Agent 的核心思想

1. 不要为所有任务构建 Agent
   - 任务复杂性
   - 任务价值
   - 关键能力验证
   - 错误成本
2. 保持简洁
   - 环境
   - 工具集
   - 系统提示
3. 像 Agent 一样思考
