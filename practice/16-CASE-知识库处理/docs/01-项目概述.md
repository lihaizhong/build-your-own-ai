# 知识库处理项目 - 项目概述

## 项目简介

知识库处理项目是一个完整的RAG（检索增强生成）知识管理系统，专注于知识库的智能优化、质量监控和版本管理。该项目基于BM25检索算法和大语言模型技术，为RAG系统提供全面的知识库维护解决方案。

## 核心功能

### 1. 问题生成与检索优化
- 为知识切片自动生成多样化问题
- 支持多种问题类型（直接问、间接问、对比问等）
- 基于BM25算法的智能检索优化
- 检索效果评估和对比分析

### 2. 对话知识提取
- 从历史对话中提取有价值信息
- 智能识别知识点类型（事实、需求、问题等）
- 知识合并和去重处理
- 知识频率统计和分析

### 3. 健康度检查
- 知识完整性检查（覆盖率和缺失分析）
- 知识时效性检查（过期信息识别）
- 知识一致性检查（冲突检测）
- 综合健康度评分和改进建议

### 4. 版本管理与性能比较
- 知识库版本创建和管理
- 版本差异比较和变更追踪
- 版本性能评估和对比
- 回归测试支持

## 技术特点

### 1. BM25智能检索
- 基于词频和逆文档频率的相似度计算
- 中文分词优化（jieba）
- 停用词过滤和文本预处理
- 双索引策略（内容索引+问题索引）

### 2. 大语言模型驱动
- 智能问题生成（多样化类型）
- 对话知识提取和分类
- 健康度智能分析
- 知识合并和去重

### 3. 向量化检索支持
- FAISS向量索引构建
- text-embedding-v4模型集成
- 语义相似度计算
- 高效向量检索

### 4. 完整的质量保障
- 覆盖率、时效性、一致性三维评估
- 加权健康度评分
- 自动改进建议生成

## 技术栈

### 核心技术
- **Python 3.11+**: 主要编程语言
- **OpenAI SDK**: 大语言模型调用
- **rank_bm25**: BM25检索算法实现
- **FAISS**: 向量相似度搜索
- **jieba**: 中文分词

### API和模型
- **通义千问 (qwen-turbo-latest)**: LLM模型
- **text-embedding-v4**: 文本向量化模型（1024维）
- **阿里云DashScope**: API服务

### 数据处理
- **NumPy**: 数值计算
- **json**: 数据序列化
- **re**: 正则表达式处理

### 工具库
- **loguru**: 日志记录
- **python-dotenv**: 环境变量管理
- **dataclasses**: 数据类支持

## 性能指标

### 检索优化效果
- **问题检索准确率提升**: 相比纯内容检索提升约15-25%
- **BM25检索响应时间**: 毫秒级（小规模知识库）
- **问题生成速度**: 每个知识切片约2-3秒

### 健康度检查
- **覆盖率评分**: 0-1区间，反映知识完整性
- **新鲜度评分**: 基于时间衰减的知识时效性
- **一致性评分**: 检测知识冲突程度
- **综合健康度**: 加权平均评分

### 版本管理
- **版本创建时间**: 取决于知识库规模和embedding速度
- **差异比较**: 自动识别增删改内容
- **性能评估**: 支持准确率和响应时间对比
