# Cycle Factor版本管理说明

## 文件命名规范修正

### 问题描述
用户指出code目录下的cycle_factor预测文件没有按照[工具]_[版本号]_prediction.py的格式命名。

### 解决方案
1. **代码文件规范化**：
   - ✅ `cycle_factor_prediction_v1.py` (基础版本)
   - ✅ `cycle_factor_prediction_v2.py` (改进版本，预测一致性优化)

2. **预测结果文件规范化**：
   - ✅ `cycle_factor_v1_predictions_201409.csv` (v1版本考试格式)
   - ✅ `cycle_factor_v2_predictions_201409.csv` (v2版本考试格式)

3. **详细结果文件规范化**：
   - ✅ `cycle_factor_v1_detailed_201409.csv` (v1版本详细结果)
   - ✅ `cycle_factor_v2_detailed_201409.csv` (v2版本详细结果)

### 版本差异

#### v1.0 基础版本特点：
- 基础的周期因子建模（weekday + day因子）
- 基础置信度评估（数据质量、因子稳定性、模型拟合、预测一致性）
- 预测一致性评估标准相对宽松（变异系数0.5-2.0）
- 置信度：75.0

#### v2.0 改进版本特点：
- 改进的预测一致性检查
- 新增趋势连续性检查（检查大变化是否合理）
- 新增业务逻辑合理性检查（周末vs工作日、月初月末规律）
- 更精细的置信度评估体系
- 预测一致性评估标准调整（0.1-4.0范围，更科学的评估）
- 置信度：78.0（提升3分）

### 运行命令

```bash
# 运行v1基础版本
uv run python code/cycle_factor_prediction_v1.py

# 运行v2改进版本（推荐）
uv run python code/cycle_factor_prediction_v2.py
```

### 预测结果对比

两个版本的预测数值相同（7,644,767,533元申购，7,525,212,575元赎回），但v2版本在以下方面有改进：

1. **更科学的置信度评估**：78.0 vs 75.0
2. **更详细的预测一致性分析**：包含趋势连续性和业务逻辑检查
3. **更精细的质量控制**：对预测异常进行更严格的检查

### 文件位置

- **代码文件**: `code/`
- **预测结果**: `prediction_result/` (考试提交格式)
- **详细结果**: `user_data/` (包含因子和置信度详情)
